Blocks, Procs, and Lambdas

Posted on November 11, 2014

What Is A Block?

A code block in Ruby is chunk of code that is wrapped between the 'do' and 'end' keywords. They allow us to group statements and associate them with method calls almost like parameters. One of the most common uses of blocks in Ruby would be Array enumeration. Any method that wants to take a block as a parameter can use the yield keyword to run the block at any time. If you provide a code block when you call a method, then inside the method, you can yield control to that code block - suspend execution of the method; execute the code in the block; and return control to the method body, right after the call to yield. For example:

def example
  yield
end

example {"Hello world"}
=> "Hello world"

What Is A Proc?

A Proc is a kind of block. When we create a block with Proc.new, it behaves like it's a part of the calling method when return is used within it, and returns from both the block itself as well as the calling method. For example:

def example
  Proc.new {return "Returned from the block"}.call
  return "Returned from the method"
end

example
=> "Returned from the block"

What Is A Lambda?

A Lambda is another type of block. When we create a block with a lambda, it behaves like a method when you use return and simply exits the block, handing control back to the calling method. Notice that it doesn't return all the way from the calling method. This is the fundamental difference between lambdas and procs, and for this reason lambdas are used more often. For example:

def example
  lambda {return "Returned from the block"}
  return "Returned from the method"
end

example
=> "Returned from the method"
