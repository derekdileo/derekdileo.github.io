Arrays vs Hashes?

Posted on October 21, 2014

What is an Array?

An array is an ordered integer-indexed collection of objects. That's a fancy way of saying that we can take objects and put them together in order, and keep their position in the same order, and we can refer to those objects by their positions. That's what that integer-indexed part is. We can say give me the first object, give me the fifth object, and so on, because they are going to be indexed according to what position they hold. One thing to remember is that arrays starting index is always 0.

Example:

fruits = ["apple", "banana", "orange"] #make an array of fruits
=> ["apple", "banana", "orange"]

fruits[0] #returns value for index of 0
=> "apple"

fruits[2] #returns value for index of 2
=> "orange"

fruits[fruits.length - 2] #returns value for second to last index
=> "banana"

What is a Hash?

Hashes are an unordered, object-indexed collection of objects. Instead of being indexed by their position and keeping track of the number, we keep track of them using an object. This is called a key-value pair. We have one object that's the 'key' and it references a second object which is the 'value'. The best example of a hash would be a dictionary because it's full of key-value pairs. In this case a key would be the 'word' and the value would be the 'definition' of that word.

Example:

dictionary = {
              "car" => "has four wheels",
              "motorcycle" => "has two wheels",
              "boat" => "has zero wheels"
            }

dictionary["motorcycle"] #returns the value of this key
=> "has two wheels"

Arrays vs Hashes?

So notice, first of all, that where an array is an ordered collection, a hash is an unordered collection. That's a very important difference. When we want to preserve the order and the order matters to us, we want to use an array. When it's not so much the order that matters, but we want the convenience of having that label, and we want to be able to refer to things by label instead, we use a hash.
