Enumerable#cycle

Posted on October 31, 2014

What is Enumerable?

Ruby's Enumerable module can be simply put as something that can be counted. If you look at the documentation for the Array class, String class, Hash class, Range class, etc. You will find that Enumerable is an included module. This is called a mixin. Because we are talking about something that can be counted, mixing in the Enumerable module allows us to add functionality like sort, collect, and search to any class.

Some of the methods made available by Ruby's awesome Enumerable module are #sort, #detect, #select, #reject, #collect, #inject, #cycle, and #group_by. I am going to be examining #cycle in more detail.

How does the #cycle method work?

When we call #cycle on an object and pass it a block, it will cycle through each element in the collection a given number of times calling the block each time. We can specify how many cycles we would like to do by passing an argument to #cycle. If we don't specify it will cycle forever.

>> array = ["one", "two", "three"]
>> array.cycle(2) { |i| puts i }
one
two
three
one
two
three
=> nil

If we pass in a negative number or the collection is empty, it will do nothing. It will return nil if the loop has finished without getting interrupted.

>> array.cycle(-2) { |i| puts i }
=> nil

If no block is given, an enumerator is returned instead.

>> cycle_enumerator = array.cycle
=> #<Enumerator: ["one", "two", "three"]:cycle>

At this point you can use Enumerator#next to get the next element of the cycle as many times as you want.

>> cycle_enumerator.next
=> "one"
>> cycle_enumerator.next
=> "two"
>> cycle_enumerator.next
=> "three"
>> cycle_enumerator.next
=> "one"
