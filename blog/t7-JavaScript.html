Ruby classes vs. JavaScript constructor functions

Posted on November 21, 2014

Differences?

The main difference between the two languages is that Ruby is class based, whereas JavaScript is not. However, it's possible to get class like behaviour in JS using constructor functions. If we want to create an object in Ruby, we say something like:

def Person
  def initialize(name)
    @name = name
  end

  def talk
    "My name is #{@name}"
  end
end

max = Person.new('Maxwell Mears')

max.talk # =>"My name is Maxwell Mears"

Classes In Ruby

In Ruby, every object has an associated class. Classes are a place to define methods, which give objects their behaviour. When we create an instance of an object, ruby protects that objects data.

Each instance of the 'Person' class has access to all the methods defined in that class, as well as instance variables through method calls.

Constructor Functions In JavaScript

Constructor functions provide a way to define behaviour which is shared by all new objects that are created by that function. We use a constuctor function with the 'Function.prototype' to share behavior between objects. This technique simulates the same behavior as Ruby classes. This looks something like:

function Person(name) {
  this.name = name;
}

Person.prototype.talk = function() {
  return "My name is" + this.name;
}

var john = new Person("John Smith");

john.talk();   // prints "My name is John Smith"

In summary, properties on JavaScipt objects are publicly accessible. Ruby uses objects inherited from Class to create new objects, whereas JS can either define literal objects, or invoke constructor functions to create new objects. JavaScript also allows for the creation of new objects with shared behaviour via Function.prototype
